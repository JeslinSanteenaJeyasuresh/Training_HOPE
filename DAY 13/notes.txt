ARRAY vs ARRAYLIST (Java)

1. Size:
- Array: Fixed size
- ArrayList: Dynamic (resizable)

2. Type:
- Array: Can store primitives & objects
- ArrayList: Stores objects only (use wrapper classes like Integer)

3. Syntax:
- Array: int[] arr = new int[5];
- ArrayList: ArrayList<Integer> list = new ArrayList<>();

4. Access:
- Array: arr[index]
- ArrayList: list.get(index)

5. Add/Remove:
- Array: Not easy (manual work)
- ArrayList: Easy (add(), remove())

6. Performance:
- Array: Faster
- ArrayList: Slightly slower (due to resizing & objects)

7. Methods:
- Array: Limited
- ArrayList: Many built-in methods (add, remove, size, etc.)

Use:
- Array → fixed size, better performance
- ArrayList → flexible size, easy operations

ARRAYLIST RESIZING & ARRAY SIZE

1. Formula:
- newSize = old + (old / 2)
- Used by ArrayList (not arrays)
- Increases size by 50%

Example:
10 → 15 → 22 → 33 ...

2. Array:
- No default size
- Must define manually
- Example: int[] arr = new int[10];

3. ArrayList:
- Default capacity = 10
- Grows automatically using 1.5x formula

4. Key Difference:
- Array → fixed size
- ArrayList → dynamic resizing

ARRAY MUTABILITY

1. Array is NOT immutable
- Elements can be modified

2. Limitation:
- Size is fixed
- Cannot add or delete elements

3. Allowed:
- arr[i] = value

4. Not Allowed:
- Increasing or decreasing size

5. Key:
